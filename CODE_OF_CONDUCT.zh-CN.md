# 行为准则

## 1. 我们的承诺

我们希望这个仓库成为一个可以对 skill 设计、评估方法和真实样例进行高强度讨论，但不会把技术分歧变成人身冲突的协作空间。

在所有项目空间中，我们承诺：
- 直接、尊重地沟通
- 用证据挑战观点，而不是攻击人
- 重视安全、隐私和负责任披露
- 尊重新人、专家以及不同的工作方式

## 2. 期望行为

- 在 Issue、PR、Review、Discussion 中给出具体、基于证据的反馈。
- 可以强烈质疑方法论、skill、评审报告或样例，但不要攻击贡献者本人。
- 发生分歧时，说明 reasoning、前提和 trade-off。
- 发现不安全内容、secret 泄露或危险指导时，按负责任方式反馈。
- 修改治理文档或双语文档时，尽量保持中英文同步。
- 优先提交高信号内容，而不是灌水、挑衅或低质量堆砌。

## 3. 不可接受的行为

- 骚扰、侮辱、威胁、歧视或人身攻击。
- 持续性的恶意指控，例如在没有证据的情况下指责他人造假或不诚实。
- 蓄意搅乱讨论、钓鱼、围攻、刷屏或低信号 spam。
- 不负责任地公开 secrets、隐私数据、exploit payload 或危险操作说明。
- 用未经审阅、无人负责的 AI 批量内容刷满讨论区。
- 任何违反 GitHub 规则或适用法律的行为。

## 4. 适用范围

本行为准则适用于：
- 仓库中的 Issue、PR、Review、Discussion、评论以及关联样例
- 与本项目直接相关、且参与者明显以项目身份发言的公开交流
- 围绕本仓库的方法论、评估、治理和输出样例展开的讨论

## 5. 如何报告

如果你遭遇或看到不可接受行为：

1. 对于不敏感的一般协作问题，请开一个带 `[CoC]` 前缀的 Issue，并提供事实时间线。
2. 对于敏感问题，请要求非公开跟进，不要在公开页面贴出私人细节。
3. 如果问题同时涉及 secrets、exploit 细节或危险内容，请按 [SECURITY.md](SECURITY.md) 处理，而不是在公开页面贴全量信息。

有帮助的报告信息包括：
- 发生位置
- 相关链接或截图
- 事实背景
- 你希望的处理结果

## 6. 执行原则

维护者会基于现有证据处理报告，并尽量在 3 个工作日内给出初步响应。

执行原则：
- 公平：基于证据判断
- 隐私：只披露必要信息
- 适度：轻微问题可先警告或要求修改；严重或重复问题可升级处理
- 允许强分歧：本仓库允许高强度技术争论，但不接受辱骂和攻击

## 7. 可能的处理措施

根据严重程度，维护者可以：
- 发出警告
- 要求修改或删除不当内容
- 锁定讨论串或限制参与
- 临时暂停参与资格
- 永久禁止参与

## 8. 与安全问题的关系

本仓库包含会影响真实 AI 行为的 skill、样例和辅助脚本。

如果某个协作问题同时构成安全风险或披露风险，维护者可能同时依据本准则和 [SECURITY.md](SECURITY.md) 处理。

## 9. 参考说明

本准则参考了 Contributor Covenant 的常见做法，并根据本仓库“skill 方法论 + 示例 + 评审 + 输出样例”的协作模式进行了调整。
