# 设计说明总览

这一部分用于详细解释 `awesome-skills` 中各个 skill 的设计过程，并结合 skill 的具体内容说明它的设计逻辑、为什么要这样设计、解决了哪些问题，以及有哪些主要的设计亮点。

`rationale/` 这一层位于 [bestpractice/](../bestpractice/README.zh-CN.md) 的通用方法论和 [skills/](../skills/index.md) 的可执行 skill 文件之间，作用是把原本隐含在 `SKILL.md` 里的设计思路、关键取舍和问题意识单独展开讲清楚。

每份设计说明通常会回答：

- 这个 skill 具体在解决什么问题
- 为什么它的流程、门禁、references 和输出契约要这样组织
- 它在规避哪些常见替代方案或失败模式
- 最主要的设计亮点是什么
