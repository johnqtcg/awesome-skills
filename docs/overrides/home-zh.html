Skill 列表

21 个生产级 Skills

每个 skill 均配有设计说明文档、量化评审报告和回归测试。

Pipeline 后端开发

go-makefile-writer 为 Go 项目设计或重构根目录 Makefile，统一 fmt / test / lint / build 门禁 git-commit 提交前 secret 扫描 + 质量门禁，自动生成规范化 commit 消息 create-pr 带多项预检门禁和结构化正文的高质量 PR 创建流程 go-ci-workflow Go 项目 GitHub Actions CI：Make 驱动、本地一致、分层门禁 go-code-reviewer 缺陷优先的自动化代码审查——聚焦真实 bug 和回归风险，而非风格备注 security-review 可利用性优先的安全审查：鉴权、输入、依赖、容器全覆盖

Testing 测试与验证

unit-test 表驱动测试，专项追踪边界值、映射丢失与并发边界 tdd-workflow 有证据的 TDD：Red→Green→Refactor 流程 + 风险路径覆盖 api-integration-test 真实运行时配置、显式门禁、超时 / 重试安全与失败诊断 e2e-test 关键用户路径的 E2E 测试，兼顾 CI 集成与可靠性 fuzzing-test Go fuzz 测试，含适用性门禁——拒绝低价值目标，避免无效模糊测试 thirdparty-api-integration-test 外部 API 契约验证，含显式执行门禁与超时控制 systematic-debugging 修复前必须完成根因分析——禁止猜测式调试循环

Research 搜索与调研

google-search 查询分类、证据链构建、交叉验证与可复用检索串 deep-research 有来源支撑的深度调研，含内容抽取与反幻觉检查

Docs 文档与规划

writing-plans 有证据支撑的实施计划，含依赖图与写后审查循环 update-doc 文档漂移检测与有证据支撑的 README 同步 readme-generator 项目形态检测 + 基于证据的 README 生成，覆盖服务 / 库 / CLI / monorepo tech-doc-writer Runbook、API 文档、RFC——类型分类、受众分析、防过时规则

Tools 工具自动化

yt-dlp-downloader 格式探测式下载：视频、播放列表、音频提取、字幕与鉴权内容 local-transcript 本地加速 ASR + 中文规范化、分段与多格式导出

快速入门

三步上手第一个 Skill

01

浏览并选择一个 skill

从上方列表中找一个与你工作场景匹配的 skill，或从 google-search 入手。 阅读配套的设计说明可以帮助理解背后的设计决策。

02

复制 skill 目录

# 项目级安装（仅当前项目）
cp -r skills/<名称> .claude/skills/<名称>

# 个人级安装（所有项目）
cp -r skills/<名称> ~/.claude/skills/<名称>
03

Claude Code 自动激活

任务匹配时 Claude Code 会自动调用 skill，无需手动配置。 如需深入了解方法论，从这里开始： 最佳实践 →
